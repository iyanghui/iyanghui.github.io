前言

由于项目中上传视频后需要接收VOD的回调，但是奇怪的是一直收不到请求；另外前端也遇到收到了回复(有数据),但是客户端不能读取内容的问题。

几个常用的HTTP Header标头

Accept

  • 作用： 浏览器端可以接受的媒体类型,例如：Accept: text/html 代表浏览器可以接受服务器回发的类型为text/html也就是我们常说的html文档, 如果服务器无法返回text/html类型的数据,服务器应该返回一个<span style="color: red">406错误</span>(non acceptable)
  • 通配符 * 代表任意类型,例如 Accept: */* 代表浏览器可以处理所有类型,(一般浏览器发给服务器都是发这个)。但是我自己测试的是，在$.ajax中，如果不申明dataType,客户端是不能获取到返回的数据(response 200 OK)。

contenttype

前台主要的contenttype为下面三种：（需要注意的是请求头中datatype与contenttype的区别，datatype是要求浏览器返回的数据格式，这里的contenttype指的是提交的数据格式）

  • multipart/form-data类型主要是上传文件时用到；
  • application/x-www-form-urlencoded类型主要是提交k-v时用到，当然这种方法也可以将json设置在v中提交json数据；
  • application/json类型主要是传递json数据用到，层次比较深的数据；
